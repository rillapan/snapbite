## Problem Statement

Fitur Scan Makanan di SnapBite saat ini menggunakan simulasi random (`Math.random()` + `setTimeout`). User upload foto, loading singkat, lalu muncul hasil acak dari `foods.json`. Tidak ada AI sungguhan. Tujuan: ganti simulasi dengan identifikasi AI nyata menggunakan Google Gemini Vision API, sepenuhnya gratis tanpa kartu kredit.

## What Makes This Cool

AI beneran mengenali makanan dari foto — bukan random. Tapi dengan twist arsitektur yang cerdas: Gemini hanya bertugas sebagai **classifier** (foto → nama makanan), sementara seluruh konten budaya (sejarah, asal daerah, lokasi) tetap disajikan dari `foods.json` yang sudah dikurasi manusia. Ini lebih cepat, lebih murah token, dan kontennya lebih akurat daripada minta AI generate semuanya.

## Constraints

- $0 biaya — pakai Gemini 2.0 Flash free tier (1500 req/hari, tanpa kartu kredit)
- API key **tidak boleh** terekspos ke browser (NEXT_PUBLIC) — harus server-side
- Dependensi baru: `@google/generative-ai` (official Google SDK, ~50KB)
- Harus tetap ada fallback simulasi jika Gemini gagal (jaringan putus, kuota habis)
- Gambar harus di-resize di client sebelum dikirim — max 512px, max 500KB (Next.js body limit default 4MB, tapi foto HP bisa 5-15MB)
- Next.js 16 App Router, TypeScript

## Premises

1. Gratis = $0, tidak butuh kartu kredit, cukup untuk masa demo — AGREED
2. AI hanya perlu klasifikasi (nama makanan), konten kultural tetap dari `foods.json` — AGREED
3. API key disimpan di `.env.local` server-side saja, tidak expose ke browser — AGREED

## Approaches Considered

### Approach A: Client-side Gemini (minimal viable)
- Effort: S | Risk: Medium
- Ubah `handleScan()` di `scan/page.tsx` saja, panggil Gemini langsung dari browser
- Cepat (1-2 jam) tapi API key terbaca di browser source code
- Oke untuk demo localhost, tidak aman untuk production/deploy

### Approach B: Next.js API Route + Gemini (ideal architecture) ← DIPILIH
- Effort: M | Risk: Low
- Buat `app/api/scan/route.ts` — server-side route yang menerima gambar, call Gemini, kembalikan hasil
- API key aman di `.env.local`
- Fallback ke random jika Gemini error

### Approach C: Transformers.js in-browser (zero API key)
- Effort: L | Risk: High
- Model 100-200MB download pertama kali
- Tidak tahu "rendang", "gudeg", "pempek" by name tanpa fine-tuning
- Over-engineering untuk demo kuliah

## Recommended Approach

**Approach B — Next.js API Route + Gemini 2.0 Flash**

### Response Schema (TypeScript)

```ts
// app/api/scan/route.ts → return shape
type ScanResponse =
  | { status: "found";    food: Food }          // cocok di foods.json
  | { status: "notFound"; suggestedName: string } // Gemini tahu tapi belum di DB
  | { status: "fallback"; food: Food }          // Gemini error → random
  | { status: "error";    message: string }     // hard error (tidak seharusnya terjadi)
```

### Arsitektur

```
User → upload foto (HP bisa 5-15MB)
     → frontend resize ke max 512px (canvas API, sebelum base64)
     → frontend convert ke base64 (<500KB setelah resize)
     → POST /api/scan { imageBase64: "data:image/jpeg;base64,..." }
     → app/api/scan/route.ts
         → AbortController timeout 10 detik
         → kirim ke Gemini 2.0 Flash (server-side, API key aman):
           prompt berisi nama semua makanan dari foods.json (generate dari data, bukan hardcode)
         → Gemini balas: "rendang"
         → Cocokkan dengan foods.json
         → Jika cocok: return { status: "found", food }
         → Jika tidak cocok: return { status: "notFound", suggestedName }
         → Jika timeout/error: return { status: "fallback", food: randomFood }
     → frontend terima hasil
     → status "found"/"fallback" → tampilkan UI hasil (sudah ada)
     → status "notFound" → tampilkan banner "Makanan ini belum ada di database kami" + suggestedName
```

### Prompt Strategy (Kunci Keberhasilan)

Prompt yang buruk: *"Analisis foto ini dan berikan informasi lengkap tentang makanan..."*
→ Gemini akan hallucinate, jawaban panjang, susah diparse

Prompt yang benar (list makanan di-generate dari `foods.json` saat runtime, bukan hardcode):
```ts
const foodNames = foods.map(f => f.nama.toLowerCase()).join(", ");
const prompt = `Kamu adalah food classifier untuk makanan Indonesia.
Lihat foto ini dan jawab HANYA dengan nama makanan dalam bahasa Indonesia,
huruf kecil, satu kata atau dua kata maksimal.
Makanan yang ada di database kami: ${foodNames}.
Jika nama makanan dalam foto ada di daftar di atas, jawab persis namanya.
Jika tidak ada di daftar tapi kamu tahu namanya, jawab nama itu.
Jika tidak yakin atau bukan makanan, jawab: "tidak dikenal"
JANGAN tambahkan penjelasan apapun.`;
```
Catatan: generate `foodNames` dari `foods.json` — jika database bertambah, prompt otomatis ikut update.

### Matching Strategy

Setelah Gemini return `"rendang"` (sudah `.trim().toLowerCase()`):
1. **Slug exact match** → `foods.find(f => f.slug === geminiResult)` — cek slug dulu (paling presisi)
2. **Nama exact match** → `foods.find(f => f.nama.toLowerCase() === geminiResult)` — jika slug tidak cocok
3. **Nama contains** → `foods.find(f => f.nama.toLowerCase().includes(geminiResult))` — fallback partial (note: cek apakah `geminiResult` ada di dalam `nama`, bukan sebaliknya — ini mencegah "nasi" cocok dengan dua item)
4. Jika multiple match pada step 3 → ambil yang pertama (prioritas urutan di `foods.json`)
5. Tidak ada yang cocok → `{ status: "notFound", suggestedName: geminiResult }`

### Cara Dapat API Key (Gratis)

1. Buka: https://aistudio.google.com
2. Sign in dengan Google account
3. Klik "Get API Key" → "Create API Key"
4. Copy, paste ke `.env.local`: `GEMINI_API_KEY=your_key_here`
5. Tidak perlu billing, tidak perlu kartu kredit

## Open Questions

1. Apakah dosen presentasi butuh koneksi internet di ruang kelas? (Jika offline, pastikan fallback simulasi berjalan)
2. Berapa banyak makanan yang akan ditambahkan ke `foods.json` sebelum presentasi? Makin banyak → makin besar kemungkinan Gemini match.

## Success Criteria

- Upload foto rendang → tampil halaman Rendang (bukan random)
- Upload foto gudeg → tampil halaman Gudeg
- Jika foto tidak dikenal → muncul pesan "belum ada di database", bukan crash
- Jika Gemini gagal (network error) → fallback ke simulasi random, tetap fungsional
- Tidak ada API key yang tampil di browser DevTools → Network tab
